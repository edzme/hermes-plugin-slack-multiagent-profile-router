# Hermes Slack multi-agent profile router

Run multiple Hermes profiles as distinct Slack agents in one workspace without
slash-command collisions.

Each Hermes profile uses:

- its own Slack app;
- its own `xoxb-` bot token and `xapp-` Socket Mode token;
- its own gateway service;
- one unique root command, such as `/support-agent` or `/product-agent`.

The profile is selected by the Slack app/token pair and gateway process. Inside
that already profile-bound process, this plugin translates the unique root
command to Hermes' existing `/hermes <subcommand>` handler.

The Slack app name, bot-user display name, Assistant description, setup text,
handoff messages, and the agent's Slack-facing self-identity also default to
the profile name. “Hermes Agent” remains the name of the underlying software,
not the public Slack persona.

For example:

```text
/support-agent model
  -> Slack app "support-agent"
  -> gateway started with --profile support-agent
  -> internal /hermes model handler
```

This is not a central router and does not multiplex several profiles behind one
Slack app.

## Requirements

- Hermes Agent with the Git plugin manager (`hermes plugins install`)
- A separate Slack app and token pair for every Hermes profile
- Socket Mode enabled on each Slack app

The plugin subclasses Hermes' bundled Slack adapter and wraps its setup
callback. It does not modify or copy Hermes core code. Because it relies on a
small private adapter surface, run the smoke tests after upgrading Hermes.

## Install

Install and enable the plugin independently for every profile:

```bash
hermes --profile support-agent plugins install \
  edzme/hermes-plugin-slack-multiagent-profile-router --enable

hermes --profile product-agent plugins install \
  edzme/hermes-plugin-slack-multiagent-profile-router --enable
```

By default, the profile name becomes both the Slack app/bot name and the unique
root command. Optional profile environment variables are:

```dotenv
SLACK_COMMAND_NAME=support-agent
SLACK_BOT_NAME=support-agent
SLACK_BOT_DESCRIPTION=Customer support agent for Slack
```

`SLACK_COMMAND_NAME` must be 1-32 lowercase letters, digits, hyphens, or
underscores and must begin with a letter or digit.

## Set up Slack

Run Hermes' normal interactive gateway setup:

```bash
hermes --profile support-agent gateway setup
```

The plugin wraps the bundled Slack setup callback, so the manifest written to
the path shown by the wizard is profile-aware. For `support-agent`, it contains:

- app name `support-agent`;
- bot display name `support-agent`;
- Slack-facing agent identity `support-agent`;
- exactly one slash command: `/support-agent`;
- the current scopes, events, Socket Mode, and Assistant settings generated by
  the installed Hermes version.

Create the Slack app from that manifest, then supply that app's unique `xoxb-`
and `xapp-` tokens to the setup wizard. Repeat with a different Slack app and
token pair for every other profile.

To regenerate the correct manifest without running interactive setup:

```bash
hermes --profile support-agent slack-profile-manifest --write
```

To use Slack's newer Agent messaging experience while retaining the
profile-specific identity and single root command:

```bash
hermes --profile support-agent slack-profile-manifest --agent-view --write
```

`--agent-view` changes the Slack messaging surface; Slack describes that
choice as irreversible after applying the manifest. Generating the JSON alone
does not change the Slack app.

You can override its display metadata:

```bash
hermes --profile support-agent slack-profile-manifest --write \
  --name support-agent \
  --description "Customer support agent for Slack"
```

Do not use stock `hermes slack manifest` for a multi-profile Slack app. That
stock command intentionally emits Hermes' generic command set. The
profile-aware command above emits the single unique root command.

After configuring or updating the app, restart its gateway:

```bash
hermes --profile support-agent gateway restart
```

## Update

Update each installed checkout independently, then restart its gateway:

```bash
hermes --profile support-agent plugins update slack-platform
hermes --profile support-agent gateway restart
```

Regenerate and reapply the Slack app manifest after a Hermes or plugin update.
Slack may request that the app be reinstalled when scopes or events change.

## Development

Run the focused test suite inside a Hermes development environment:

```bash
pytest -q tests/test_slack_profile_router.py
```

The tests cover:

- command-name validation;
- single-command manifest generation;
- profile-root translation to `/hermes`;
- Socket Mode handler registration;
- profile-aware manifest generation during interactive Slack setup.
